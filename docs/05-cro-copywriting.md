# CRO And Copywriting System

## Core Conversion Strategy

Kuboqss must convert cold TikTok/Snapchat/Facebook traffic by answering four questions quickly:

1. What is this and why should I care?
2. Can I trust this store?
3. Is the result relevant to my problem?
4. Is ordering safe and easy?

## Above The Fold Formula

For every product page:

- Badge: `اختيار كوبوكس`
- Headline: outcome + audience
- Subheadline: simple mechanism + COD trust
- Rating row: stars + review count
- Offer selector
- CTA
- Microtrust line

Example:

```text
اختيار كوبوكس للعناية اليومية
حل بسيط لروتين جمال أوضح بدون تعقيد
منتج مختار بعناية للمرأة الليبية، مع طلب سريع ودفع عند الاستلام.
```

CTA:

```text
اختاري العرض واطلبي الآن
```

Microtrust:

```text
الدفع عند الاستلام - سنتواصل معك لتأكيد الطلب قبل الشحن
```

## Home Page CRO

Home should not be a generic catalog. It must establish Kuboqss as the authority.

Must include:

- Premium hero.
- COD trust strip.
- "How we choose products" section.
- Featured campaigns/products.
- Social proof.
- UGC/video placeholders.
- FAQ.
- Delivery reassurance.

Suggested hero:

```text
كوبوكس: اختيارات جمال موثوقة داخل ليبيا
منتجات عناية مختارة بعناية، تجربة طلب سريعة، ودفع عند الاستلام بدون تعقيد.
```

CTA:

```text
شاهدي اختياراتنا
```

Secondary CTA:

```text
كيف نختار منتجاتنا؟
```

## Social Proof

Use placeholders until real reviews exist. Do not fake exact claims that imply verified customers if not true.

Acceptable placeholder labels:

- "تجارب عميلاتنا قريباً"
- "مساحة مخصصة لفيديوهات وتجارب العميلات"
- "سيتم إضافة تقييمات حقيقية بعد أول الطلبات"

After real sales:

- Add name initials, city, product, and date.
- Example: `م.س - طرابلس`
- Use short reviews in natural Arabic.

Review template:

```text
الطلب كان سهل والتأكيد وصلني بسرعة. المنتج وصلني مرتب والتغليف شكله جميل.
```

## Scarcity And Urgency

Use honest urgency only.

Good:

```text
الكمية المتاحة لهذا العرض محدودة حسب مخزون المورد.
```

```text
العرض قابل للتغيير بعد انتهاء الدفعة الحالية.
```

Avoid fake:

- "باقي 3 فقط" if not connected to real inventory.
- Countdown timers that always reset.

## Offer Copy

Use three offer cards:

1. Trial:

```text
قطعة واحدة
مناسبة للتجربة الأولى
```

2. Best value:

```text
قطعتان
الأكثر طلباً - قيمة أفضل
```

3. Routine/family:

```text
ثلاث قطع
أفضل اختيار للروتين أو المشاركة
```

Default selected offer should usually be the middle offer to increase AOV.

## Product Page Emotional Structure

Use this sequence:

1. Name the pain.
2. Make the user feel understood.
3. Introduce Kuboqss selection.
4. Explain why the product is relevant.
5. Show proof/mechanism.
6. Reduce risk with COD and confirmation.
7. Ask for the order.

Example:

```text
إذا كنتِ تشعرين أن روتين العناية صار متعباً ونتائجه غير واضحة، فاختيار المنتج المناسب يختصر عليك الكثير.
```

```text
في كوبوكس نركز على منتجات سهلة الاستخدام، مفهومة، ومناسبة للطلب داخل ليبيا مع الدفع عند الاستلام.
```

## FAQ Questions

Use on product pages:

- هل الدفع عند الاستلام؟
- متى يتم تأكيد الطلب؟
- هل المنتج مناسب لكل الناس؟
- كيف أستعمل المنتج؟
- متى تظهر النتيجة؟
- هل يمكنني طلب أكثر من قطعة؟
- ماذا لو أدخلت رقم الهاتف خطأ؟

Answers should be honest and short.

## Confirmation And Delivery Rate Copy

Thank-you page and checkout microcopy must prepare the user:

```text
بعد إرسال الطلب، سيقوم فريقنا بالتواصل معك لتأكيد التفاصيل قبل الشحن. يرجى إبقاء الهاتف متاحاً حتى لا يتأخر طلبك.
```

```text
الدفع يكون عند الاستلام فقط.
```

## Libya-Specific Language

Use:

- `داخل ليبيا`
- `الدفع عند الاستلام`
- `تأكيد الطلب`
- `فريقنا سيتواصل معك`
- `طرابلس، بنغازي، مصراتة، سبها...` only where delivery actually covers these cities.

Do not overclaim national delivery unless true.

## High AOV Rules

- Make two-piece offer the default.
- Cross-sell only relevant products.
- Use "complete the routine" language.
- Show free delivery threshold only if business supports it.
- Do not discount everything. Premium brand should use bundles, not constant coupons.
